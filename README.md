## Output:
FitLife -UX Project Report & Prototype Overview This document summarizes the UX work for the FitLife Health Tracking App and explains the prototype (website) included with this submission.

Problem Statements:
Fitness enthusiasts need a way to easily track daily activities and progress because existing apps are either too complex or fragmented, which results in inconsistent usage and loss of motivation. Health-conscious users need a way to receive personalized workout and diet insights because current apps lack tailored recommendations, which results in poor engagement and low goal completion rates.

Research Methods & Justification Selected methods:

User Interviews: 

to discover motivations, routines, and blocker behaviors. - Surveys: to validate patterns and measure prevalence.

Usability Testing: 

to iterate on flows and prototypes for clarity and efficiency. Justification: Combining qualitative (interviews, usability testing) and quantitative (surveys) gives both depth and scale insights. Usability testing ensures the prototype is effective.

Personas Persona A - Arjun Mehta Age: 28 | Software Engineer | Bangalore Goals: Maintain consistent workouts get concise feedback Behaviors Checks quicksummaries, uses smartwatch. Pain Points: Too many apps, inconsistent tracking. Motivation: Better health and productivity. Persona B - Sneha Reddy Age: 35 | Marketing Manager | Hyderabad Goals: Manage weight and nutrition. Behaviors: Follows reminders and meal logs. Pain Points: Gets discouraged when progress isn't visible. 

Motivation:

Positive reinforcement and goal visualization.
Prototype Scope & Pages The provided prototype is a simple web app demonstrating the FitLife flows and Ul patterns. Pages included: Welcome / Splash - Login / Signup - Dashboard (summary + quick actions) - Activity /Product Listing (workouts) - Checkout / Final Action (subscribe / goal completion)
